## What is Mi Tienda?

Mi Tienda is a full-stack payment checkout application that demonstrates a production-ready integration with the **Wompi** payment gateway. Users can browse products, enter their card and delivery details, and receive real-time payment results — all secured through Wompi's client-side tokenization so raw card data never touches your server.

<Info>
  Card data is tokenized directly in the browser via Wompi's API. Your backend only ever receives a card token — never raw card numbers.
</Info>

## Key features

<CardGroup cols={2}>
  <Card title="Secure tokenization" icon="shield-check">
    Credit card data is tokenized client-side via Wompi. The card number never reaches your backend.
  </Card>
  <Card title="Real-time status polling" icon="rotate">
    After payment submission, the frontend polls the transaction status every 2.5 seconds (up to 12 attempts) until a final result is returned.
  </Card>
  <Card title="Clean Architecture backend" icon="layer-group">
    NestJS backend with strict domain/application/infrastructure separation, Result pattern, and 125 unit tests at ~99% coverage.
  </Card>
  <Card title="Persistent checkout state" icon="floppy-disk">
    Redux Toolkit tracks the full checkout flow across page reloads using localStorage persistence.
  </Card>
  <Card title="Auto-seeded products" icon="box">
    Three demo products are automatically inserted into the database on first startup — no manual setup needed.
  </Card>
  <Card title="Swagger API docs" icon="file-code">
    Interactive API documentation available at `/api/docs` when the backend is running.
  </Card>
</CardGroup>

## Tech stack

| Layer | Technology |
|-------|------------|
| Backend | NestJS 11, TypeScript, TypeORM, PostgreSQL 14 |
| Frontend | React 19, Vite 7, Redux Toolkit 2, React Router 7, React Hook Form 7 |
| Payments | Wompi (client-side tokenization + server-side processing) |
| Validation | class-validator (backend), React Hook Form + Luhn algorithm (frontend) |
| Testing | Jest (backend, 125 tests), Vitest (frontend) |
| Documentation | Swagger/OpenAPI at `/api/docs` |

## Checkout flow overview

```
[1] Product selection
        ↓
[2] Card + delivery form
        ↓
[3] Order summary & confirmation
        ↓
[4] Tokenization (Frontend → Wompi)
        ↓
[5] Payment processing (Frontend → Backend → Wompi)
        ↓
[6] Status polling (until resolved)
        ↓
[7] Result (APPROVED / DECLINED / ERROR)
```
