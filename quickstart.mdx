## Prerequisites

Before you begin, make sure you have the following installed and available:

- **Node.js 18+** — both the backend and frontend require Node 18 or later.
- **PostgreSQL 14+** — the backend connects to a local or remote PostgreSQL instance.
- **Wompi sandbox credentials** — register at [sandbox.wompi.co](https://sandbox.wompi.co) to obtain your public key, private key, events key, and integrity key.

<Info>
  All four Wompi keys are available in the sandbox merchant dashboard under the **Developers** section.
</Info>

## Setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/your-org/payment-checkout-app.git
    cd payment-checkout-app
    ```
  </Step>

  <Step title="Install backend dependencies">
    <CodeGroup>
      ```bash npm
      cd backend
      npm install
      ```

      ```bash yarn
      cd backend
      yarn install
      ```

      ```bash pnpm
      cd backend
      pnpm install
      ```
    </CodeGroup>
  </Step>

  <Step title="Configure the backend environment">
    Copy the example file and fill in the required values:

    ```bash
    cp .env.example .env
    ```

    Open `backend/.env` and set the following variables:

    ```bash
    # PostgreSQL connection
    DB_HOST=localhost
    DB_PORT=5432
    DB_USER=your_postgres_user
    DB_PASS=your_postgres_password
    DB_NAME=checkout_db

    # Wompi credentials (from sandbox.wompi.co)
    PAYMENT_PUBLIC_KEY=pub_stagtest_...
    PAYMENT_PRIVATE_KEY=prv_stagtest_...
    PAYMENT_EVENTS_KEY=stagtest_events_...
    PAYMENT_INTEGRITY_KEY=stagtest_integrity_...
    PAYMENT_API_URL=https://api-sandbox.co.uat.wompi.dev/v1

    # Server
    PORT=3001
    FRONTEND_URL=http://localhost:5173

    # Fees in cents (COP)
    BASE_FEE_IN_CENTS=300000
    DELIVERY_FEE_IN_CENTS=200000
    ```

    <Warning>
      `DB_USER` and `DB_PASS` have no defaults — the backend will fail to start if they are blank. The database `checkout_db` must exist before starting; create it with `createdb checkout_db` if needed.
    </Warning>
  </Step>

  <Step title="Start the backend">
    ```bash
    npm run start:dev
    ```

    The API starts on `http://localhost:3001/api`. On first run, the seed script automatically inserts three products if the `products` table is empty:

    | Product | Price |
    |---------|-------|
    | Audífonos Bluetooth Pro | $150,000 COP |
    | Teclado Mecánico RGB | $250,000 COP |
    | Mouse Inalámbrico Ergonómico | $80,000 COP |
  </Step>

  <Step title="Install frontend dependencies">
    Open a new terminal tab:

    <CodeGroup>
      ```bash npm
      cd frontend
      npm install
      ```

      ```bash yarn
      cd frontend
      yarn install
      ```

      ```bash pnpm
      cd frontend
      pnpm install
      ```
    </CodeGroup>
  </Step>

  <Step title="Configure the frontend environment">
    ```bash
    cp .env.example .env
    ```

    Open `frontend/.env`:

    ```bash
    VITE_API_URL=http://localhost:3001/api
    VITE_PAYMENT_API_URL=https://api-sandbox.co.uat.wompi.dev/v1
    VITE_PAYMENT_PUBLIC_KEY=pub_stagtest_...
    ```

    <Note>
      `VITE_PAYMENT_PUBLIC_KEY` must match the public key used in the backend. This is the key the browser uses to call Wompi's tokenization endpoint directly — no private key is ever sent to the browser.
    </Note>
  </Step>

  <Step title="Start the frontend">
    ```bash
    npm run dev
    ```

    The SPA is available at `http://localhost:5173`.
  </Step>

  <Step title="Verify everything is working">
    Check all three services:

    | Service | URL | Expected |
    |---------|-----|----------|
    | Backend API | `http://localhost:3001/api/products` | JSON array of products |
    | Swagger UI | `http://localhost:3001/api/docs` | Interactive API documentation |
    | Frontend | `http://localhost:5173` | Product selection page |

    To run a test payment, select a product, fill in card details using a [Wompi sandbox test card](https://docs.wompi.co/docs/en/sandbox), and complete the checkout flow.
  </Step>
</Steps>

## Running tests

```bash
# All 125 unit tests
npm test --prefix backend

# With HTML coverage report
npm run test:cov --prefix backend

# A specific module
npx jest --testPathPattern="products" --prefix backend
```

## Production build

```bash
# Backend
npm run build --prefix backend && npm run start:prod --prefix backend

# Frontend
npm run build --prefix frontend   # outputs to frontend/dist/
```

<Tip>
  The frontend build runs TypeScript type checking before emitting output. Fix any type errors reported by `npm run build` before deploying.
</Tip>
