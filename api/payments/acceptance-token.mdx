title: "Get Acceptance Token"
description: "Retrieves the Wompi acceptance token required before tokenizing a card."
---

## `GET /api/payments/acceptance-token`

Fetches the current acceptance token from Wompi. The frontend **must** call this endpoint and pass the returned token along with every payment request.

<Info>
  Wompi requires users to accept its terms and conditions before a card can be tokenized. The acceptance token encodes that consent. It is generated by Wompi and is short-lived — always fetch a fresh token immediately before presenting the payment form.
</Info>

<Note>
  This endpoint is a thin proxy to the Wompi API. The token it returns comes directly from Wompi's `/merchants/:publicKey` response and is not stored by the backend.
</Note>

## Request

No parameters required.

```bash
curl -X GET https://your-api.example.com/api/payments/acceptance-token
```

## Response

<ResponseField name="success" type="boolean">
  Always `true` on a successful response.
</ResponseField>

<ResponseField name="data" type="object">
  Acceptance token data.

  <Expandable title="properties">
    <ResponseField name="acceptanceToken" type="string">
      The Wompi acceptance token. Pass this value as the `acceptanceToken` field when calling `POST /api/transactions`.
    </ResponseField>
  </Expandable>
</ResponseField>

```json
{
  "success": true,
  "data": {
    "acceptanceToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
```

## Error Responses

| Status | `success` | Description |
|--------|-----------|-------------|
| `502`  | `false`   | Wompi's API was unreachable or returned an unexpected error. |
| `500`  | `false`   | Internal server error. |

```json
{
  "success": false,
  "message": "No se pudo obtener el token de la pasarela de pago"
}
```
