## `GET /api/products`

Returns all products in the catalog. Each product includes its current stock status and a formatted price string ready for display.

<Info>
  On first startup, the database is seeded with 3 demo products: **Audífonos Bluetooth Pro** ($150.000 COP), **Teclado Mecánico RGB** ($250.000 COP), and **Mouse Inalámbrico Ergonómico** ($80.000 COP).
</Info>

## Request

No parameters required.

```bash
curl -X GET https://your-api.example.com/api/products
```

## Response

Returns a success envelope with an array of product objects.

<ResponseField name="success" type="boolean">
  Always `true` on a successful response.
</ResponseField>

<ResponseField name="data" type="array">
  Array of product objects.

  <Expandable title="Product object properties">
    <ResponseField name="id" type="string (UUID)">
      Unique identifier for the product.
    </ResponseField>

    <ResponseField name="name" type="string">
      Display name of the product.
    </ResponseField>

    <ResponseField name="description" type="string | null">
      Product description. May be `null` if not set.
    </ResponseField>

    <ResponseField name="priceInCents" type="integer">
      Product price in Colombian peso centavos (COP). For example, `15000000` represents $150.000 COP.
    </ResponseField>

    <ResponseField name="formattedPrice" type="string">
      Human-readable price string (e.g. `"$150.000"`). Pre-formatted for display.
    </ResponseField>

    <ResponseField name="imageUrl" type="string | null">
      URL of the product image. May be `null` if not set.
    </ResponseField>

    <ResponseField name="stockUnits" type="integer">
      Number of units currently available in stock.
    </ResponseField>

    <ResponseField name="hasStock" type="boolean">
      `true` if at least one unit is available; `false` if the product is out of stock.
    </ResponseField>
  </Expandable>
</ResponseField>

{

  "data": [

## Error Responses

| Status | `success` | Description |
|--------|-----------|-------------|
| `500`  | `false`   | Internal server error — unexpected failure when querying the database. |

```json
{
  "success": false,
  "message": "Internal server error"
}
```
